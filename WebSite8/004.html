ADD Product to Cart

Shop Tab

When the ADD button is clicked in the product table, the id of the product is sent to this action method "ADD".

This method is contained inside the "ProductTables" controller.

This id parameter is used to find the name of the product and then to determine whether the cart contains one of these products.

If it doesn't, an object for the cart table is created, assaigned the appropriate values and is added to the cart table.

If the product already exists in the cart table then the quantity is incremented.

SQL Design Issue

Missing Primary/Foreign Key Constraint

When I created the database I forgot to set up a primary and foreign key constraint between these tables.

The cart table should have a ProductID parameter rather than the Name, Price and Image parameters.

This way the product table would act as a kind of look-up or reference table for the cart table.

When I remake this project I will rectify this.
